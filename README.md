# Sentinel-1 Explorer

Browser-side rendering of **Sentinel-1 SAR** imagery on Development Seed's
`deck.gl-raster` / `deck.gl-geotiff` GPU pipeline. No tile server, no derived
data, no hosting: the browser searches a public STAC catalog, opens each
Cloud-Optimized GeoTIFF over HTTP Range, reprojects it on the client, and renders
it on the GPU.

**Live demo:** https://kentstephen.github.io/s1-amplitude-explorer/

The app opens on the Lena River Delta (Siberia) with the 2018-01-31 scene loaded
as a dual-pol composite, so a fresh visit lands on imagery, not a blank basemap.

## What it does

- **Two render modes.**
  - *Amplitude*: single-polarization grayscale dB (`10*log10(intensity)`) then a
    two-way dB stretch then gamma. The dramatic relief / texture look.
  - *Composite*: dual-pol false colour from VV, VH, and the VV/VH ratio. Two
    palettes: *natural* (R=VV, G=VH, B=ratio) and *cb-safe* (a red-green
    colourblind-safe mapping using blue/yellow plus luminance instead of R/G).
- **Polarization toggle:** VV (cleanest terrain look) or VH (volume scattering).
- **Search and load:** search the current view or a drawn AOI over a date window,
  step through candidate dates ranked by coverage, then load the most-complete
  mosaic or a single date. Place search is built in.
- **Tuning:** two-thumb dB range slider and a gamma slider, with per-handle reset.
- **Export:** capture a PNG of the current map (no UI chrome).
- **Sessions:** save the look, dates, AOI, and map view to the browser and
  restore on reload; reset returns to defaults.

The control panel is organized as collapsible Find / Look / View folders on one
scrolling page.

## Data

Primary source is **Earth Search `sentinel-1-grd`** (AWS Open Data) via the
STAC search at `https://earth-search.aws.element84.com/v1/`. Measurement COGs are
read over the CORS-open virtual-hosted S3 endpoint, no token and no
requester-pays. The runtime STAC client is in `web/src/stac.ts`.

**Product: IW GRD**, not RTC (RTC flattens the terrain shading that makes relief
pop) and not SLC (complex InSAR data).

## Reprojection and seams

S1 GRD COGs are per-scene, so the app warps each one to web mercator on the
client using its GCP grid (`web/src/gcp.ts`, `gcpTileset.ts`,
`GcpMultiCOGLayer.ts`) rather than relying on a pre-tiled basemap. At high
latitudes (the Lena Delta sits at ~72N) the warp is strong, so the reprojection
mesh error budget is relaxed by zoom band (`maxErrorForZoom` in `App.tsx`) far
enough that the refiner converges and adjacent frames register, which closes the
inter-frame seam. Mosaic scene counts and AOI span are capped for smooth panning.

## Run

```bash
cd web
npm install
npm run dev        # http://localhost:5455
```

`npm run build` produces the static site; `npm run typecheck` runs `tsc`.

## Deploy

Pushes to `main` build `web/` and deploy to GitHub Pages via
`.github/workflows/deploy.yml` (Pages source set to "GitHub Actions"). The Vite
`base` is `/s1-amplitude-explorer/` to match the repo, so the site serves from
`https://kentstephen.github.io/s1-amplitude-explorer/`.

## Stack

- [`@developmentseed/deck.gl-raster`](https://github.com/developmentseed/deck.gl-raster)
  for the GPU shader pipeline (dB, `LinearRescale`, `Colormap`, the dual-pol
  composite), and `@developmentseed/deck.gl-geotiff` for COG streaming
  (`MosaicLayer` / `COGLayer` / `MultiCOGLayer`).
- `@developmentseed/proj` for the client-side GCP reprojection.
- deck.gl 9.3, luma.gl 9.3, maplibre-gl 5, react-map-gl 8, React 19, Vite 6.

> The deck.gl-raster `0.7` line moves fast and the shader-pipeline shape can
> change across minors. Check the latest from the Dev Seed repos before upgrading.

## Origin

Forked from
[`sentinel-2-cog-deckgl-raster`](https://github.com/kentstephen/sentinel-2-cog-deckgl-raster),
with the Sentinel-2 spectral pipeline and all elevation/terrain code removed and
the S1 dB / dual-pol pipeline and per-scene GCP warp built in its place. The
sibling project `deckgl-raster-mapterhorn-s2` holds the terrain work that was not
carried over. Inherited S2 design notes live under
[`docs/reference-s2-inherited/`](./docs/reference-s2-inherited/); data-source
findings are in [`docs/RESEARCH.md`](./docs/RESEARCH.md).
