# Sentinel-1 Amplitude Explorer

Browser-side rendering of **Sentinel-1 SAR amplitude** as dramatic monochrome
imagery over mountain ranges and deserts, using Development Seed's
`deck.gl-raster` / `deck.gl-geotiff` GPU pipeline. No tile server, no derived
data, no hosting: the browser opens each Cloud-Optimized GeoTIFF over HTTP Range
and renders it client-side.

> Status: **scaffold**. Forked from
> [`sentinel-2-cog-deckgl-raster`](https://github.com/kentstephen/sentinel-2-cog-deckgl-raster)
> (the two-way-slider COG/STAC app), with the Sentinel-2 spectral pipeline and
> all elevation/terrain code dropped. The S1 data wiring and the dB display
> shader are the work to be done. See [`docs/RESEARCH.md`](./docs/RESEARCH.md)
> for the feasibility and data-source findings, and `docs/PLAN.md` once it exists.

## Why Sentinel-1 (and not ICEYE-class data)

We looked at ICEYE's open data and want that dramatic monochrome SAR look over
wide terrain. ICEYE Spotlight is sub-1 m; Sentinel-1 IW GRD is ~10 m. So S1 will
**not** match ICEYE for close-up crispness. But for **wide mountain/desert
scenery**, S1's side-looking geometry produces strong relief shading (bright
foreslopes, dark radar shadows) across a 250 km swath in one frame, which is
exactly the dramatic regional look we're after. Coverage and relief shading beat
crispness here. Full reasoning: [`docs/RESEARCH.md`](./docs/RESEARCH.md).

Key decisions coming out of the research:

- **Product: IW GRD**, not RTC. RTC flattens the terrain shading that makes
  mountains pop; GRD keeps it. (And not SLC, which is complex data for InSAR.)
- **Polarization: VV** default (cleanest terrain look), VH as a toggle.
- **Display: dB stretch** (`10*log10(intensity)`) then linear rescale then
  grayscale or colormap. This replaces the S2 NDVI math as the centerpiece shader.
- **No elevation.** The terrain/mapterhorn layer from the sibling project is
  intentionally not carried over.

## Data source (planned)

Primary path: **Earth Search `sentinel-1-grd`** COGs on AWS public storage
(`https://earth-search.aws.element84.com/v1/`). Same STAC-POST shape as the S2
Earth Search client this was forked from (`web/src/earthSearchStac.ts.ref` is
the S2 reference). Ambitious second track: **EOPF GeoZarr** via deck.gl-raster's
`ZarrLayer` (the ESA "Sentinel into Zarr" effort behind
[explorer.eopf.copernicus.eu](https://explorer.eopf.copernicus.eu/)). Trade-offs
in [`docs/RESEARCH.md`](./docs/RESEARCH.md).

## Run

```bash
cd web
npm install
npm run dev        # http://localhost:5455
```

The app currently still runs the **inherited Sentinel-2** wiring until the S1
data path is built. It's a working deck.gl-raster COG/STAC base to fork from,
not yet an S1 app.

## What was inherited vs. dropped

Carried over from `sentinel-2-cog-deckgl-raster`:

- deck.gl-raster 0.7 + deck.gl-geotiff 0.7 + @developmentseed/geotiff 0.7 COG
  streaming, MosaicLayer/COGLayer/MultiCOGLayer.
- The map shell, place search, draw-AOI tool, load scoreboard, colormap set, and
  the **dual-thumb range slider** (Radix) for two-way stretch.
- The runtime STAC client shape (`web/src/stac.ts`) and the Earth Search variant
  (`web/src/earthSearchStac.ts.ref`).

Dropped (vs. the siblings):

- All Sentinel-2 spectral indices (NDVI/NDWI) and the multiband composite path.
  S1 amplitude is single-band.
- All elevation/terrain code (mapterhorn PMTiles, ElevatedCOGLayer, terrain
  loaders). This fork is 2D amplitude only.

The S2 design docs are preserved under
[`docs/reference-s2-inherited/`](./docs/reference-s2-inherited/) for the
deck.gl-raster patterns (seams, perf knobs, the MultiCOG pattern) they document.

## Stack

- [`@developmentseed/deck.gl-raster`](https://github.com/developmentseed/deck.gl-raster)
  — GPU shader pipeline (`LinearRescale`, `Colormap`, and a new `ZarrLayer` for
  the GeoZarr track).
- [`@developmentseed/deck.gl-geotiff`](https://github.com/developmentseed/deck.gl-raster/tree/main/packages/deck.gl-geotiff)
  — `MosaicLayer` / `COGLayer`.
- deck.gl 9.3 + luma.gl 9.3 + maplibre-gl 5 + react-map-gl 8 + React 19.

> The deck.gl-raster `0.7` line moves fast and the shader-pipeline shape changes
> across minors. **Check the latest from the Dev Seed repos before developing** —
> the pins here came over from the S2 fork.
